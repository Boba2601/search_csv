# CSV Search Tool

Простой Python-скрипт для поиска строк в CSV-файле по ключевому слову или тексту. Найденные строки сохраняются в отдельный файл, при этом заголовок таблицы сохраняется всегда, а полностью одинаковые (дублирующиеся) строки автоматически пропускаются.

## Возможности

- 🔍 Поиск текста в любой части строки (без учёта регистра)
- 📄 Сохранение результата в отдельный CSV-файл
- 🧾 Автоматическое сохранение заголовка таблицы в результат — чтобы было понятно, какая колонка за что отвечает
- 🚫 Пропуск полностью одинаковых строк (дубликатов) — в результат попадают только уникальные совпадения
- 🌍 Поддержка разных кодировок (utf-8, cp1251 и др.)
- 🔧 Поддержка разных разделителей (`,`, `;`, табуляция и т.д.)
- 💬 Интерактивный режим — скрипт сам спрашивает нужные данные, ничего не нужно передавать через командную строку

## Требования

- Python 3.6+
- Дополнительные библиотеки не требуются (используется только стандартная библиотека `csv`)

## Установка

```bash
git clone https://github.com/your-username/csv-search-tool.git
cd csv-search-tool
```

## Использование

Запустите скрипт:

```bash
python search_csv.py
```

Скрипт последовательно запросит:

1. **Путь к исходному CSV файлу** — проверяется, что файл существует
2. **Путь для сохранения результата** — например, `result.csv` (по умолчанию, если оставить пустым)
3. **Текст/слово для поиска** — обязательное поле
4. **Кодировку файлов** — необязательно, по умолчанию `utf-8`
5. **Разделитель CSV** — необязательно, по умолчанию `,`

### Пример работы

```
=== Поиск строк в CSV файле ===

Введите путь к исходному CSV файлу: data.csv
Введите путь для сохранения результата (например, result.csv): found.csv
Введите текст/слово для поиска: Москва
Кодировка файлов (Enter — по умолчанию utf-8): 
Разделитель CSV (Enter — по умолчанию ','): 

Готово!
Найдено уникальных строк: 12
Пропущено дубликатов: 3
Результат сохранён в: found.csv
```

## Как это работает

1. Скрипт открывает исходный CSV файл и построчно читает его.
2. Первая строка (заголовок таблицы) сразу копируется в файл результата — это позволяет видеть названия колонок.
3. Каждая последующая строка проверяется на наличие искомого текста (без учёта регистра).
4. Если строка содержит искомый текст, скрипт проверяет, не встречалась ли уже точно такая же строка ранее.
5. Уникальные совпадения записываются в выходной файл, повторяющиеся — пропускаются.
6. В конце выводится статистика: сколько строк найдено и сколько дубликатов пропущено.
