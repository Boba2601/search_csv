# CSV Search Tool

Python-скрипт для работы с CSV-файлами: поиск строк по одному или нескольким словам, а также удаление полностью одинаковых (дублирующихся) строк по всему файлу. Работает в интерактивном режиме — просто запускаете скрипт, и он сам спрашивает всё необходимое.

## Возможности

- 🔍 Два режима работы:
  1. **Поиск строк** по одному или нескольким словам
  2. **Удаление дубликатов** по всему файлу без поиска
- 🔗 Поиск по нескольким словам с выбором логики:
  - **ИЛИ** — строка подходит, если найдено хотя бы одно из слов
  - **И** — строка подходит, только если найдены все слова сразу
- 🧾 Заголовок таблицы всегда сохраняется первой строкой в результате — сразу понятно, какая колонка за что отвечает
- 🚫 Автоматический пропуск полностью одинаковых строк (дубликатов)
- 🌍 Поддержка разных кодировок (utf-8, cp1251 и др.)
- 🔧 Поддержка разных разделителей (`,`, `;`, табуляция и т.д.)
- 💬 Полностью интерактивный режим — ничего не нужно передавать через командную строку

## Требования

- Python 3.6+
- Дополнительные библиотеки не требуются (используется только стандартная библиотека `csv`)

## Установка

```bash
git clone https://github.com/your-username/csv-search-tool.git

```

## Использование

Запустите скрипт:

```bash
python search_csv.py
```

Скрипт спросит:

1. **Режим работы**:
   - `1` — поиск строк по слову/словам
   - `2` — просто убрать дубликаты по всему файлу (без поиска)
2. **Путь к исходному CSV файлу** — проверяется, что файл существует
3. **Путь для сохранения результата** — например, `result.csv` (по умолчанию, если оставить пустым)

Если выбран режим **поиска**, дополнительно спросит:

4. **Слово(-а) для поиска** — можно ввести несколько слов через запятую
5. Если слов больше одного — **логику поиска**: ИЛИ (хотя бы одно слово) или И (все слова сразу)

В обоих режимах в конце спросит:

6. **Кодировку файлов** — необязательно, по умолчанию `utf-8`
7. **Разделитель CSV** — необязательно, по умолчанию `,`

## Как это работает

1. Скрипт открывает исходный CSV файл и построчно читает его.
2. Первая строка (заголовок таблицы) сразу копируется в файл результата.
3. **В режиме поиска** — каждая последующая строка проверяется на наличие искомых слов (без учёта регистра), с учётом выбранной логики (ИЛИ/И).
4. **В режиме удаления дубликатов** — поиск не выполняется, проверяются все строки файла подряд.
5. В обоих режимах: если строка совпала с условием (или режим — просто дедуп), скрипт проверяет, не встречалась ли уже точно такая же строка ранее.
6. Уникальные строки записываются в выходной файл, повторяющиеся — пропускаются.
7. В конце выводится статистика: сколько строк сохранено и сколько дубликатов пропущено.
